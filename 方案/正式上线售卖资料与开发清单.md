# Agent 时代教程正式上线售卖资料与开发清单

更新时间：2026-04-27

## 1. 当前版本基线

当前已经具备的是一个第一版资料站雏形：

- 卡密登录入口：用户输入卡密后进入站内。
- 登录后全站内容暂时全部可见：不做单章节解锁，不做作业提交、学习监督、成绩统计。
- 左侧导航：首页、学习路径、资料库、实战案例、工具模板、收藏夹。
- 页面内容：目前还是前端假数据占位。
- 云函数：`auth-function` 已能验证卡密并返回登录 token，本地和腾讯云控制台测试通过。

上线售卖前，还需要把“假数据占位”替换成真实内容，把“环境变量卡密”升级成数据库卡密系统，并补齐购买、交付、后台管理和基础合规页面。

## 2. 需要你提供给我的资料清单

| 类别 | 你需要提供什么 | 是否适合存腾讯云文档数据库 | 推荐存储方式 |
|---|---|---:|---|
| 产品定位 | 产品名称、目标用户、核心卖点、适合人群、不适合人群、交付形式 | 是 | 文档数据库存配置 |
| 售卖方案 | 价格、套餐名称、有效期、是否限设备、是否限次数、是否支持退款 | 是 | 文档数据库存套餐配置 |
| 卡密规则 | 卡密格式、批次名称、每批数量、有效期、设备绑定规则、失效规则 | 是 | 文档数据库存卡密记录，但卡密建议存哈希 |
| 教程目录 | 一级模块、章节标题、排序、预计学习顺序、章节简介 | 是 | 文档数据库存结构化目录 |
| 教程正文 | 每节教程的标题、正文、步骤、提示词、注意事项、常见问题 | 是 | 短正文可存文档数据库；长正文建议 Markdown 文件放云存储，数据库存索引 |
| 实战案例 | 案例标题、项目目标、步骤、假数据、最终效果、源码说明 | 是 | 文档数据库存元数据和正文索引 |
| 工具模板 | 提示词模板、检查清单、项目模板、复制按钮内容 | 是 | 文档数据库存文本模板 |
| 下载资料 | PDF、Word、Excel、ZIP、源码包、截图素材、视频文件 | 不建议直接存 | 腾讯云云存储，文档数据库只存文件 ID、名称、大小、类型 |
| 封面和图片 | 课程封面、案例截图、资源封面、品牌图标 | 不建议直接存 | 腾讯云云存储，数据库存 URL 或 fileID |
| 销售页面文案 | 首页卖点、购买按钮文案、FAQ、适合谁、交付清单 | 是 | 文档数据库或前端静态配置 |
| 法务资料 | 用户协议、隐私政策、退款规则、免责声明、版权声明 | 是 | 文档数据库或静态 Markdown |
| 客服资料 | 联系方式、微信/邮箱、售后流程、常见问题回复模板 | 是 | 文档数据库 |
| 发票和主体信息 | 公司/个体信息、收款主体、发票规则、备案信息 | 部分适合 | 非敏感配置可进数据库，证照文件放云存储 |

结论：腾讯云文档数据库适合存“结构化数据、文本内容、目录、卡密状态、订单状态、用户访问状态”。大文件不要直接塞进文档数据库，PDF、ZIP、图片、视频、源码包都应放腾讯云云存储，数据库只保存文件的 `fileID`、访问 URL、大小、类型、可见状态。

## 3. 建议的数据集合设计

| 集合名 | 用途 | 关键字段 |
|---|---|---|
| `site_config` | 站点基础配置 | `brandName`、`logoUrl`、`supportContact`、`homeCopy` |
| `plans` | 售卖套餐 | `name`、`price`、`durationDays`、`deviceLimit`、`isActive` |
| `card_keys` | 卡密管理 | `keyHash`、`batchId`、`planId`、`status`、`expiresAt`、`usedAt`、`boundDevices` |
| `users` | 访问用户 | `nickname`、`deviceIds`、`planId`、`createdAt`、`lastLoginAt` |
| `sessions` | 登录会话 | `userId`、`tokenHash`、`deviceId`、`expiresAt`、`revokedAt` |
| `modules` | 学习路径模块 | `title`、`summary`、`sort`、`status` |
| `lessons` | 教程章节 | `moduleId`、`title`、`content`、`contentFileId`、`sort`、`tags` |
| `projects` | 实战案例 | `title`、`summary`、`steps`、`assets`、`sort` |
| `templates` | 工具模板 | `title`、`body`、`category`、`copyText`、`sort` |
| `resources` | 资料下载 | `title`、`type`、`fileId`、`fileUrl`、`size`、`category` |
| `orders` | 订单记录 | `orderNo`、`planId`、`amount`、`status`、`paidAt`、`cardKeyId` |
| `audit_logs` | 关键操作日志 | `action`、`userId`、`deviceId`、`ip`、`createdAt` |

卡密不要明文存储。正式版建议：生成卡密后只保存哈希值，用户提交卡密时云函数计算哈希再比对。这样即使数据库泄漏，也不会直接泄漏可用卡密。

## 4. 还需要继续开发的功能清单

### P0：正式售卖必须完成

- 卡密数据库化：从 `ACCESS_KEYS` 环境变量升级为 `card_keys` 集合。
- 卡密登录云函数：支持卡密校验、有效期、状态、设备绑定、重复登录策略。
- HTTP 路由：把 `auth-function` 暴露为前端可调用接口，例如 `/api/agent-auth`。
- 前端接入真实登录接口：配置 `VITE_AUTH_FUNCTION_URL`，移除本地 demo 卡密逻辑或只在开发环境保留。
- 内容接口：新增读取模块、教程、资源、模板、案例的云函数。
- 前端内容动态化：把当前假数据改为从数据库读取。
- 资源下载保护：下载 PDF/ZIP/源码前校验登录 token，避免公开链接被转发滥用。
- 云存储资料上传：把 PDF、ZIP、图片、源码包放到腾讯云云存储。
- 订单和卡密发放流程：支付成功后生成或发放卡密。
- 错误状态页面：无效卡密、过期卡密、设备超限、接口失败、资源不存在。
- 部署配置：生产域名、环境变量、CORS、安全域名、HTTPS、构建发布流程。

### P1：上线体验建议完成

- 购买页/售卖页：展示卖点、目录、交付清单、价格、FAQ。
- 卡密兑换页：用户购买后输入卡密进入资料站。
- 用户中心：显示当前套餐、有效期、绑定设备、退出登录。
- 教程详情页：展示真实教程正文、步骤、提示词复制、相关资源。
- 实战案例详情页：案例目标、步骤、素材、源码下载。
- 资源库详情页：分类、搜索、文件大小、下载按钮。
- 工具模板详情页：模板分类、变量说明、一键复制。
- 全站搜索：按标题、标签、资源类型搜索。
- 后台管理页面：管理教程、资源、模板、案例、卡密批次、订单。
- 卡密批量生成：支持批次、数量、有效期、套餐绑定、导出 CSV。
- 基础运营数据：只做后台运营用的访问量、卡密使用量、订单量，不放在用户学习界面里。

### P2：增长和运营增强

- 优惠码/活动价。
- 续费/升级套餐。
- 内容更新日志。
- 收藏夹真实持久化。
- 最近打开真实记录。
- 客服入口和问题反馈。
- 防刷限制：IP 限频、接口限频、异常卡密尝试记录。
- 数据备份和导出。
- 管理员权限分级。

## 6. 推荐上线顺序

1. 先补齐资料：目录、教程正文、资源文件、售卖文案、价格规则。
2. 建数据库：`plans`、`card_keys`、`modules`、`lessons`、`resources`、`templates`、`projects`。
3. 改登录：云函数从数据库校验卡密，生成真实 session。
4. 改内容：前端从数据库读取内容，不再使用假数据。
5. 做下载保护：资料文件放云存储，下载前校验卡密登录状态。
6. 做售卖页和购买流程：先可以人工发卡，后续再接自动支付发卡。
7. 做后台管理：先做卡密和内容管理，订单管理可后置。
8. 部署生产：域名、HTTPS、CORS、安全域名、备份和监控。

## 7. 我下一步最需要你给的材料

为了最快进入可售卖版本，优先给我这 8 类资料：

1. 产品正式名称和一句话卖点。
2. 售价、套餐、有效期、设备限制规则。
3. 第一版真实目录：模块、章节、顺序。
4. 每个章节的正文或原始素材。
5. 需要下载的 PDF、ZIP、表格、源码包。
6. 购买页文案：适合谁、能拿到什么、常见问题。
7. 退款规则、用户协议、隐私政策基础文本。
8. 卡密发放规则：人工发卡还是支付后自动发卡。

第一版建议先做“卡密售卖 + 全站通行证”的简单模型，不做复杂会员等级，也不做单内容解锁。这样能最快上线验证付费意愿。
